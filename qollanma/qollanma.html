E-PEDAGOG LOYIHASI
Himoya uchun maxsus texnik, amaliy va arxitektura qo'llanmasi

1. Maqsad va Dolzarblik

E-Pedagog loyihasi oliy ta'lim muassasasi xodimlarining faoliyatini to'liq raqamlashtirishga qaratilgan platforma hisoblanadi. Professur-o'qituvchilarga an'anaviy qog'ozli yillik hisobotlar (fayllar va papkalar hajmini ko'paytiruvchi rasmiy hujjatlar) o'rniga, bulutli texnologiyalarga asoslangan raqamli monitoring qilish va real vaqtda tasdiqlash vositasini taqdim etadi.

2. Dasturiy Texnologiyalar (Stack)

Frontend (Interfeys)

  • React 18 (Vite orqali)
  • TypeScript (Asosiy til)
  • Tailwind CSS (Dizayn va stil)
  • Framer Motion (UI animatsiyalari)

Backend & Database

  • Supabase (Backend as a Service)
  • PostgreSQL (Relyatsion ma'lumotlar bazasi)
  • Supabase Auth (Tizimga kirish xavfsizligi)
  • Supabase Storage (Hujjat va fayllar buluti)

3. Tizimli Arxitektura Grafigi

   [FOYDALANUVCHI (O'qituvchi / Mudir)] 
           |
       (Brauzer / SPA Interfeys)
           |
   { React.js / Tailwind CSS } 
           |   (Real vaqt so'rovlari: JSON / REST / WebSockets)
           V
   [ SUPABASE CLOUD BACKEND ]
      ├── Supabase Auth (Parollar, Rollar va sessiyalar)
      ├── PostgreSQL DB (Jadvallar, hisobotlar, qaydlar)
      └── Storage Bucket (PDF, DOCX fayllar ombori)

4. Dastur Funksiyalari

  • Rolli Boshqarish: 4 xil darajadagi ko'rish huquqlari - Pedagog (kirituvchi), Tasdiqlovchi (tekshiruvchi), Tahrirlovchi (bo'lim) va Admin.
  • Avtomatik hisob kitoblar: O'quv (Bahorgi va Kuzgi) ishlarini avtomat tarzda yillik reja bilan hisob-kitob qilish, Excel ga olib chiqish imkoniyati.
  • Fayl Yuklash Tizimi: Har bir kiritilgan yutuq uchun dalil sifatida maqola, tezislar nusxalarini yoki sertifikatlarni PDF tarzida ilova qilish.
  • Tarix (Logs): Admin tomonidan xavfsizlik nazorati - qachon va kim ma'lumotni o'zgartirganini monitoring qilish.
  • Online Chat: Admin bilan muammolarni xal qilish uchun "Live Chat" tizimi.
  • Responsive Print: Yillik ish hujjatini belgilangan formatda chop etish uchun tayyor qog'oz-A4 versiya shablonini generatsiya qilish.

5. Foydalanuvchi qanday qilib darsturdan foydalanadi?

  1. Tizimga shaxsiy korporativ yoki ochiq email va parol yordamida ro'yxatdan o'tadi (LoginPage, RegisterPage).
  2. Sessiya aniqlangach, Foydalanuvchi tegishli Dashboardga yo'naltiriladi (M: PedagogDashboard).
  3. Kuzgi yoki Bahorgi semestrni tanlab "Ilmiy ishlar", "O'quv uslubiy ishlar" oynasidagi katakchalarni to'g'ridan to'g'ri (xuddi Excel kabi) tahrirlash imkoniyatidan foydalanadi (ScientificWork / MethodicalWork).
  4. Dalil uchun "Qog'oz qisqichi (attach)" ikonunu bosib tizimga fayl yuklaydi. Bular keshlanadi.
  5. "Saqlash" tugmasi bosilganda barcha o'zgarishlar table_data ko'rinishida Supabase DB-ning JSON array sifatdi saqlanadi.
  6. Yakunda, administrator/tasdiqlovchilar uchun ochiq modul orqali bular analiz qilinadi va O'qituvchining yillik reytingi yakunlanadi.
